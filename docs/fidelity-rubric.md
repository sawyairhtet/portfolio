# GNOME 49 / Fedora 43 Fidelity Rubric

This portfolio is judged as a desktop simulation, not a themed landing page. A score of 5 means the behavior, visuals, timing, and small details are convincing under close inspection. A score of 3 means it looks plausible in a screenshot. A score of 1 means it borrows a few GNOME ideas. A score of 0 means the category is effectively absent.

Primary references:

- `docs/reference/design-tokens.md`
- `docs/reference/libadwaita.md`
- `docs/reference/gnome-hig.md`
- `docs/reference/gnome-49-release-notes.md`
- `docs/reference/fedora-43.md`
- `src/styles/adwaita-tokens.css`

## General Scoring Rules

- Score each category from 0 to 5. Half points are allowed only when a category clearly sits between two levels.
- If a feature is purely decorative but should be interactive in GNOME, it cannot score above 2.
- If a visual value is not traceable to the reference docs or `src/styles/adwaita-tokens.css`, it counts against fidelity even when it looks nice.
- If a category breaks keyboard access, focus handling, or reduced-motion expectations, it cannot score above 3.
- If the implementation depends on vague "GNOME-like" styling instead of exact tokens and behaviors, it cannot score above 3.

## 1. Color Fidelity

Every interface color must be traceable to a libadwaita token or GNOME palette value. Content imagery may contain natural colors; chrome, app UI, controls, text, shadows, borders, focus, and state colors do not get that exemption.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | Uses an unrelated portfolio palette, random gradients, glassmorphism, Tailwind defaults, or brand colors as UI chrome.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 1     | Uses a blue accent and dark surfaces, but colors are mostly hard-coded and not tied to Adwaita tokens.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 2     | Major backgrounds roughly match GNOME dark/light mode, but semantic colors, borders, hover states, and inactive states are inconsistent.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 3     | Most visible chrome uses token-like colors, but scattered hex/rgb values remain in components or states.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 4     | All major surfaces, text, borders, semantic states, and accent states map to documented tokens; only a few low-risk legacy values remain.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| 5     | Every UI color outside the token file is consumed through a documented token such as `--window-bg-color`, `--view-bg-color`, `--headerbar-bg-color`, `--card-bg-color`, `--accent-bg-color`, `--accent-color`, `--destructive-color`, `--warning-color`, `--success-color`, `--border-color`, or `--shade-color`. Dark mode uses `#222226` window, `#1d1d20` view, and `#2e2e32` headerbar/sidebar where appropriate. Light mode uses `#fafafb` window and white view/headerbar surfaces. Accent color changes propagate across focus rings, selected rows, switches, app badges, and active controls without introducing new colors. |

Ruthless checks:

- `rg "#|rgb|hsl|oklch|color-mix" src css` should find token definitions or justified token usage, not component-local color invention.
- Hover, active, disabled, selected, destructive, warning, success, and focused states all need token-backed colors.
- High contrast opacity overrides must use `--border-opacity`, `--dim-opacity`, and `--disabled-opacity`.

## 2. Typography Fidelity

The UI must feel like GNOME 49, not a web portfolio. Typography should use Adwaita Sans for interface text and Adwaita Mono for code, terminal, and logs.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | Uses unrelated display fonts, marketing typography, aggressive letter spacing, or viewport-scaled text.                                                                                                                                                                                                                                                                                                                                                                                                          |
| 1     | Uses `system-ui` or Cantarell-era assumptions with inconsistent sizes and weights.                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 2     | Uses Adwaita Sans somewhere, but headings, captions, code, and controls do not match the HIG scale.                                                                                                                                                                                                                                                                                                                                                                                                              |
| 3     | Uses Adwaita Sans for most UI and Adwaita Mono for code; common labels and headings are close.                                                                                                                                                                                                                                                                                                                                                                                                                   |
| 4     | Applies the HIG/libadwaita style classes consistently: title, heading, body, caption, document, monospace, and numeric text behave predictably.                                                                                                                                                                                                                                                                                                                                                                  |
| 5     | UI defaults to Adwaita Sans 11pt, document content to Adwaita Sans 12pt, code and terminal to Adwaita Mono 11pt. The scale matches the reference: `.title-1` 181%/800, `.title-2` 136%/800, `.title-3` 136%/700, `.title-4` 118%/700, `.heading` 700, `.body` 400 with 140% line height, `.caption` 82% with 140% line height, and `.numeric` uses tabular numbers for clocks, timers, battery, and file sizes. Text never relies on negative letter spacing or hero-scale marketing type inside desktop chrome. |

Ruthless checks:

- No UI text should fall back to Inter, SF Pro, Roboto, JetBrains Mono, or custom display fonts unless the real app would.
- Text in controls must fit at 100%, 125%, and 150% browser zoom.
- Window titles, app labels, menu items, notifications, dialogs, and settings rows must use UI-scale text, not landing-page hierarchy.

## 3. Window Chrome

Windows must behave like GNOME/libadwaita windows, including focus, drag, resize, shadows, rounded corners, and headerbar behavior.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ----- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | Content appears as cards, modals, or browser panels with no desktop window model.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 1     | Has fake windows, but chrome is decorative and cannot be dragged, focused, resized, minimized, or closed reliably.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 2     | Windows can open and close, but titlebars, shadows, focus states, controls, and rounded corners are visibly web-made.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 3     | Windows can be dragged and focused; headerbar, radius, and shadow are roughly Adwaita-like.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 4     | Floating and maximized states, inactive/backdrop styling, stacking order, resize cursors, and headerbar controls are mostly faithful.                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| 5     | Headerbars use the documented 46px minimum height and Adwaita surface tokens. Floating windows use `--window-radius: 15px`; maximized and fullscreen windows use `0px` radius. Buttons and small controls respect 24px/36px control minimums and `--button_radius: 9px`. Focused windows have stronger chrome/shadow; unfocused windows use backdrop colors. Dragging from headerbars feels native, resizing has correct hit regions, double-click maximizes/restores, edge snapping works, Esc closes transient dialogs, and window controls match GNOME placement and behavior for the represented app. |

Ruthless checks:

- A window that looks right but cannot be dragged scores at most 2.
- A maximized window with rounded outer corners scores at most 3.
- Inactive windows must be visibly but subtly de-emphasized, not just left unchanged.

## 4. Motion

Motion must use libadwaita-like easing, duration, cancellation, and reduced-motion behavior. Flashy web animation is a liability.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 0     | No intentional motion, or random CSS transitions copied from generic UI kits.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 1     | Basic fades/slides exist but timing and easing are inconsistent.                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| 2     | Most transitions are short and pleasant, but curves and durations are not traceable to libadwaita tokens.                                                                                                                                                                                                                                                                                                                                                                                              |
| 3     | Common control transitions use `200ms` and Adwaita-like ease-out curves; major surfaces still feel approximate.                                                                                                                                                                                                                                                                                                                                                                                        |
| 4     | Window, overview, toast, tab, focus, button, and color-scheme transitions use documented timings and respect reduced motion.                                                                                                                                                                                                                                                                                                                                                                           |
| 5     | Button and focus transitions use `--ease-out-quad` with 200ms timing. Backdrop transitions use `--backdrop_transition`. Attention states use 150ms ease-in. Toasts use 300ms show/hide and 500ms replace behavior. Overview and workspace transitions use Adwaita ease or spring parameters rather than generic bouncy curves. Color-scheme changes complete around 250ms. All motion is interruptible, does not queue stale animations, and collapses cleanly under `prefers-reduced-motion: reduce`. |

Ruthless checks:

- Any `transition: all` in desktop chrome is suspect and must be justified.
- Animations longer than 500ms need a GNOME source or they fail the category.
- Motion must never hide state changes, delay keyboard action, or make windows feel like cards.

## 5. Activities Overview

The overview is the heart of GNOME Shell. A static app launcher is not enough.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | No Activities overview.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 1     | A launcher or overlay exists, but it is not tied to the Super key or workspace model.                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| 2     | Super opens an overlay with app icons, but windows, workspaces, search, and keyboard navigation are shallow.                                                                                                                                                                                                                                                                                                                                                                                                               |
| 3     | Overview shows open windows and app grid; Super/Esc toggles it; search is usable.                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 4     | Workspaces, app grid, search, live window previews, and keyboard navigation are mostly integrated.                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 5     | Pressing Super opens/closes Activities with the correct focus behavior. The overview shows scaled live window previews, a workspace switcher, active workspace state, and app grid. Users can click, keyboard-navigate, search, launch apps, switch windows, move windows between workspaces, and close the overview with Esc. Ctrl+Alt+arrows update the workspace strip. The overview restores focus to the correct app/window after dismissal and uses GNOME-like motion, spacing, dimming, and search field treatment. |

Ruthless checks:

- A static screenshot of workspaces scores at most 1.
- An app grid without window/workspace awareness scores at most 2.
- Super key behavior must not fight browser shortcuts or leave focus trapped.

## 6. Authentic Apps

Apps must behave like real GNOME apps, not portfolio sections wearing app chrome. The minimum app set is Files, Console, Text Editor, Calendar, Image Viewer, and Settings.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | Apps are absent or are links to portfolio sections.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 1     | Apps open as static mockups with little or no state.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 2     | Apps have plausible shells and a few buttons, but most interactions are dead ends.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 3     | Each required app has a useful core workflow and persistent enough state to survive casual testing.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 4     | Each app has menus, keyboard shortcuts, empty/loading/error states, realistic data, and app-specific behavior.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 5     | Every required app withstands close imitation testing. Files supports sidebar places, grid/list views, breadcrumbs/path entry, search popover, hidden files with transparency, cut files with dashed border and scissor cue, rename, trash, context menus, and selection behavior. Console supports prompt input, command history, scrollback, copy/paste, selection, tabs or sessions where represented, and Adwaita Mono. Text Editor supports opening/saving simulated documents, unsaved indicators, search, selection, undo/redo, and realistic headerbar actions. Calendar supports month/week navigation, today state, event popovers, event creation/editing, and keyboard navigation. Image Viewer supports opening images, zoom, pan, rotate, fit-to-window, metadata/info, and image navigation. Settings has a sidebar, search, Appearance, Displays, Keyboard, Sound, Power, Network, Accessibility, and About surfaces; toggles persist and affect the shell where applicable. |

Ruthless checks:

- Any required app that is a stub caps this category at 2.
- Any app with nonfunctional primary buttons caps this category at 3.
- Portfolio content can live inside apps only if it behaves as app data, not as a marketing page.

## 7. Keyboard Fidelity

Keyboard behavior is judged as desktop interaction. Mouse-only fidelity is not fidelity.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ----- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | No meaningful keyboard support.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| 1     | Tab focus exists incidentally, but shell/app shortcuts are missing.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 2     | A few global shortcuts work, but focus management, app shortcuts, and overlays are unreliable.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| 3     | Super, Esc, Alt+Tab, and common app shortcuts mostly work.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 4     | Global shortcuts, app shortcuts, focus restoration, roving focus, and shortcut help are consistent.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| 5     | Super opens Activities. Alt+Tab shows an app/window switcher with MRU ordering. Ctrl+Alt+Left/Right moves between workspaces. Esc closes popovers, dialogs, menus, overview, and transient surfaces in the right order. `/` starts search where GNOME apps expect it, especially Files. `?` opens shortcut help where represented. Arrow keys navigate grids, menus, calendar dates, lists, and workspace previews. Enter/Space activate focused controls. Ctrl+L, Ctrl+F, Ctrl+W, Ctrl+Q, Ctrl+N, Ctrl+S, and app-specific shortcuts work where the real app supports them. Focus rings are visible, token-colored, and never lost after a window closes. |

Ruthless checks:

- If a keyboard user can get trapped, the category scores at most 2.
- If Super does not work, the category scores at most 3.
- If focus returns to the wrong place after closing a modal or app, the category scores at most 4.

## 8. Sound Design

Sound should be subtle, optional, and GNOME-like. Silence is better than arcade UI sounds.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                     |
| ----- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | No sound support, or intrusive unrelated sounds.                                                                                                                                                                                                                                                                                                                                             |
| 1     | A few generic click sounds exist without settings, volume discipline, or browser autoplay handling.                                                                                                                                                                                                                                                                                          |
| 2     | Notification and click sounds exist but are not integrated with app state or user preferences.                                                                                                                                                                                                                                                                                               |
| 3     | Notifications, clicks, errors, and toggles have subtle sounds with a mute control.                                                                                                                                                                                                                                                                                                           |
| 4     | Sounds are restrained, mixed quietly, respect system/settings state, and are triggered only by meaningful events.                                                                                                                                                                                                                                                                            |
| 5     | Notification arrival, button activation, toggles, errors, calendar reminders, terminal bell, volume changes, and login/lock events use subtle Adwaita-style sounds. Sounds never play before user gesture permission. The Settings app can mute/adjust them, and the shell respects Do Not Disturb. Audio is short, low-volume, non-overlapping, and never used as the only state indicator. |

Ruthless checks:

- Sound that surprises the user caps the score at 2.
- Sound that cannot be disabled caps the score at 3.
- A click sound on every hover or pointer move fails the category.

## 9. Easter Eggs And Depth

Depth means authentic secondary details that reward exploration. Novelty that breaks GNOME fidelity does not count.

| Score | Criteria                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ----- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 0     | No details beyond the visible first screen.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 1     | A few decorative jokes or hidden links exist, unrelated to GNOME behavior.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| 2     | Some secondary states exist, but they are shallow or inconsistent with the desktop metaphor.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| 3     | Several discoverable details make the desktop feel lived-in: context menus, notifications, app data, and settings persistence.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 4     | The environment has many authentic secondary states that react to time, settings, files, windows, and keyboard use.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| 5     | The desktop has screenshot-worthy depth without feeling like a gimmick. Examples: Fedora 43 day/night wallpaper timing, GNOME 49 Do Not Disturb in Quick Settings, lock screen media controls, battery plugged-in-but-not-charging state, HDR brightness slider when enabled, Files hidden-file transparency and cut-file scissor treatment, realistic About dialogs, shortcut overlays, drag-to-workspace behavior, persistent recent files, calendar reminders, terminal command easter eggs that affect real app state, notifications that stack correctly, and small chrome changes between focused/unfocused/maximized/backdrop states. Every hidden detail should make the simulation feel more real. |

Ruthless checks:

- Easter eggs that use off-brand visuals, meme UI, or loud animation reduce the score.
- Depth must be discoverable through normal GNOME behavior, not only secret clicks.
- A detail counts only if it is implemented, stateful, and integrated with the rest of the desktop.

## Final Judgment Bands

| Average | Judgment                                                                                           |
| ------- | -------------------------------------------------------------------------------------------------- |
| 0.0-1.4 | Not a GNOME desktop simulation.                                                                    |
| 1.5-2.4 | GNOME-inspired skin.                                                                               |
| 2.5-3.4 | Convincing at a glance, weak under use.                                                            |
| 3.5-4.4 | Strong simulation with visible gaps.                                                               |
| 4.5-5.0 | Faithful enough that GNOME users start nitpicking tiny details instead of questioning the premise. |

To claim "most faithful GNOME 49 / Fedora 43 desktop on the web", the portfolio should average at least 4.7 and score no lower than 4 in any category.
