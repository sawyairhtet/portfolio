# Personal Portfolio

Recruiter-friendly portfolio for Saw Ye Htet, built as a Fedora/GNOME-inspired desktop experience in the browser.

## About

The current site is a React and TypeScript application built with Vite. It presents Java, Spring Boot, SQL, and frontend proof through a desktop-style shell with a boot sequence, dock, draggable windows, terminal, quick settings, notifications, and deep-linkable app routes.

## Tech Stack

- React 19
- TypeScript
- Vite
- React Router
- React Hook Form + Zod
- Vitest
- ESLint flat config
- CSS custom properties and layered component styles

## Design notes

The shell follows the GNOME 49 / libadwaita visual language (window radius, header bars, Adwaita color ramp, Cantarell, accent #3584e4). Two intentional deviations from the HIG, kept for portfolio UX:

- **Persistent dock.** Stock GNOME 49 only shows the dash inside the Activities overview; this site keeps the dash visible at all times so visitors can navigate without learning the gesture. Closer to Dash-to-Dock than vanilla GNOME.
- **Icon-plus-label buttons in body content** (About CTAs, Contact actions, Project actions). HIG prefers icon-or-label outside header bars; the combined form is kept here because the audience is recruiters scanning quickly.

## Security

- **Headers**: Netlify serves `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Strict-Transport-Security` (2-year HSTS with preload), `Permissions-Policy` (blocks camera, microphone, geolocation, interest-cohort), and a strict Content Security Policy on every response.
- **CSP `style-src 'unsafe-inline'`**: Required by Tailwind CSS v4 runtime styles and React inline `style` props. This is an accepted trade-off.
- **Contact form**: Uses a honeypot field, client-side rate limiting (8 s cooldown), and Zod schema validation. Submissions go through Formspree (the form ID is public by design).
- **External links**: All `target="_blank"` links use `rel="noopener noreferrer"`. The embedded GitHub iframe is sandboxed.
- **Dependencies**: Dependabot runs weekly for both npm and GitHub Actions. CodeQL scans JavaScript/TypeScript on every push to main.

## Updating Profile & Projects

All portfolio content lives in two config files:

- **`src/config/profile.ts`** — Name, role, headline, summary, education, email, resume path, availability, location, primary stack, and social links.
- **`src/config/data.ts`** — App definitions, projects (with tech stack, links, media), skill categories, wallpaper options, and accent colors.

To add a project, append a new `Project` object to the `PROJECTS` array in `data.ts`. Set `featured: true` for the hero card. After any content change run `npm run validate` to catch type errors.

## Deployment & Post-Deploy Verification

The site deploys automatically to Netlify when commits land on `main`.

### Manual deploy

```bash
npm run build
npx netlify deploy --prod --dir=dist
```

### Post-deploy checklist

1. Visit the root URL — confirm boot sequence and desktop load.
2. Visit `/app/about` — confirm the deep link opens the About window.
3. Open DevTools → Application → Service Workers — verify `sw.js` registered with no errors.
4. Test OG preview: paste the URL into [opengraph.xyz](https://www.opengraph.xyz/) — confirm image, title, and description.
5. Verify security headers at [securityheaders.com](https://securityheaders.com/).
6. Run Lighthouse (Performance, Accessibility, Best Practices).
7. Submit the contact form with a real email to confirm Formspree delivery.

## OG Image

The Open Graph preview image is generated by Puppeteer:

```bash
npm run generate:og    # outputs public/images/og-preview.png
```

After regeneration, optimize the PNG before committing (install `pngquant` or use ImageMagick):

```bash
# pngquant (best results)
pngquant --force --quality=85-95 --output public/images/og-preview.png public/images/og-preview.png

# ImageMagick fallback
magick public/images/og-preview.png -strip -colors 256 -define png:compression-level=9 public/images/og-preview.png
```

The image is kept as PNG (not WebP) because Facebook, LinkedIn, iMessage, and Slack link previews have inconsistent WebP support.
